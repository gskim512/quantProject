주차별 개발 계획

  • 1주차 계획
    • DB 구축 : 크롤링, 지표계산, DB적재 [크롤링 할 3가지] - 코스피, 코스닥 종목별 정보 (네이버 증권) - 각 종목별 2년간의 수정주가 정보 (네이버 증권 차트) - 각 종목별 재무제표 (FnGuide에서 제공하는 재무제표) [DB에 적재할 3가지] - 코스피, 코스닥 종목별 정보 - 각 종목별 2년간의 수정주가 정보 - 재무제표 기반으로 계산한 퀀트 지표 정보
    • 프론트엔드 : 웹페이지 뼈대(HTML+CSS) 구축 참고소스 : Newpaper 형식의 웹페이지(https://codepen.io/oliviale/pen/BaoXOOP) 큰 틀의 느낌만 참고하고, 레이아웃 구성은 내가 원하는 형태로 재구성 레이아웃은 크게 3가지 영역으로 나누었다. 주식 차트 영역 / 퀀트 정보 영역 / 사이드바 영역(퀀트 별 순위) 주식 차트의 경우 Highchart API를 활용. 간단히 종목의 날짜별 주가 정보만 입력하면 주식차트를 그릴 수 있는 간편한 API. 나머지의 영역의 경우 필요한 항목들을 부트스트랩을 활용하여 구성.

  • 2주차 계획
    • 프론트엔드 : 웹페이지 기능(자바스크립트) 구축
    • 백엔드 : 날짜별 주가 정보 API, 퀀트 정보 API, 퀀트 순위 API 개발

  • 3주차 계획
    • 프론트엔드 <-> 백엔드 <-> DB 연결 구축
    • 크롤링 자동화 구축
