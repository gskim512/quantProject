프로젝트 진행 순서

  1. DB 구축
    • 네이버 증권 페이지에서 필요한 주식 종목 크롤링(주가, 정보 등)
    • FnGuide 웹페이지에서 종목별 재무제표 크롤링
    • 크롤링 한 정보 기반으로 종목별 6가지 퀀트 지표 계산 > 위험조정수익률(누적수익률/변동성) > 밸류(PER, PBR, PCR, PSR 종합 순위) > 퀄리티1(F-Score : 수익성, 재무성과, 운영효율성을 통한 스코어) > 퀄리티2(자기자본이익률(ROE), 매출총이익, 영업활동현금흐름) > 마법공식(이율(Earnings Yield), 투하자본수익률(Return on Capital)) > 멀티팩터(밸류, 퀄리티, 누적수익률 종합 평가)
    • 종목별 주가, 6가지 퀀트 지표를 DB에 적재
    • 매일 자동 크롤링 로직 구축

  2. 프론트엔드 개발
    • 총 2개의 페이지로 구축
      • 1페이지 : 종목별 정보 제공
        종목을 검색하면 해당 종목의 차트/기본정보(현재가, 발행 수)/퀀트지표
      • 2페이지 : 각 퀀트 지표별 순위 제공(30위까지)

  3. 백엔드 개발
    • 주식 차트를 그리기 위해 종목의 일자별 주가정보를 주는 GET API 개발
    • 각 종목의 퀀트 정보를 불러오기 위한 POST API 개발
    • 퀀트 종목별 순위 정보를 불러오기 위한 GET API 개발
